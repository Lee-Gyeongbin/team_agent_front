# 실수 방지 규칙 (메모장 패턴)

## 핵심 원칙

> 잘못된 작업 → 메모장 업데이트 → 같은 실수 다시 하지 마

## 동작 방식

1. **실수 발생 시**: 작업 중 오류, 잘못된 접근, 사용자 지적이 있으면 즉시 인지
2. **메모장 기록**: 두 곳에 모두 기록
   - `~/.claude/projects/*/memory/MEMORY.md` — 자동 로드 (개인용)
   - `.claude/memory.md` — git 포함 (팀 공유용)
3. **다음 작업 전 확인**: 유사한 작업 시작 전 메모장을 참고하여 같은 실수 방지

## 기록 형식

```markdown
### [카테고리] 실수 요약
- **상황**: 어떤 작업을 하다가
- **실수**: 무엇을 잘못했는지
- **해결**: 올바른 방법은 무엇인지
- **날짜**: YYYY-MM-DD
```

## 기록 대상

- 사용자가 지적한 잘못된 코드 패턴
- 프로젝트 컨벤션 위반 (CLAUDE.md 규칙 무시)
- 반복 실패한 접근 방식
- 런타임 에러를 유발한 코드
- 잘못된 파일/경로 참조

## 기록하지 않는 것

- 단순 오타 수정
- 사용자 요구사항 변경 (실수가 아님)
- 일회성 환경 문제 (node_modules 깨짐 등)
